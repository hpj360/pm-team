# Skill: Requirement Analyzer

## 描述

需求分析辅助工具，帮助需求分析师进行需求调研、分析和文档生成。

## 功能

### 1. 需求解析

将用户的原始需求描述解析为结构化的需求项。

**输入**: 用户需求描述文本
**输出**: 结构化需求列表

### 2. 用户故事生成

根据需求生成标准格式的用户故事。

**格式**:
```
作为一个 <角色>
我想要 <功能>
以便于 <价值>
```

### 3. 优先级分析

使用MoSCoW方法对需求进行优先级排序。

**优先级定义**:
- Must have: 核心功能，必须有
- Should have: 重要功能，应该有
- Could have: 期望功能，可以有
- Won't have: 暂不实现

### 4. 需求验证

检查需求的完整性和一致性。

**验证项**:
- [ ] 需求是否明确
- [ ] 需求是否可测量
- [ ] 需求是否可验证
- [ ] 需求是否有验收标准
- [ ] 需求是否有优先级

## 使用示例

```json
{
  "action": "analyze",
  "input": "用户需要一个支持多语言切换的博客系统"
}
```

## 输出格式

```json
{
  "requirements": [
    {
      "id": "REQ-001",
      "title": "多语言支持",
      "description": "系统支持中文、英文等多种语言切换",
      "priority": "Must have",
      "userStory": "作为一个国际用户，我想要切换语言，以便于阅读我熟悉的语言内容",
      "acceptanceCriteria": [
        "用户可以在设置中选择语言",
        "系统默认语言为浏览器语言",
        "语言切换后页面内容实时更新"
      ]
    }
  ]
}
```

## 配置

```json
}
```
