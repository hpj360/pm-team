## 1. 产品概述

### 1.1 项目背景

网络安全红方业务需要处理海量安全相关文件，包括但不限于：渗透测试报告、漏洞扫描结果、流量捕获文件（PCAP）、日志文件、恶意样本、配置文件、目标资产信息等。这些文件是红方业务的核心数据资产，其管理效率直接影响安全测试工作的质量和效率。

随着红方业务的不断拓展，面临的主要挑战包括：文件数量庞大且增长迅速，传统管理方式难以满足存储需求；文件类型多样，缺乏统一的解析和分析手段；信息检索困难，无法快速定位所需资料；文件之间缺乏关联分析，难以形成完整的情报链条；敏感文件缺乏有效的安全管控，存在数据泄露风险。

因此，亟需建设一个专业的红方文件汇聚平台，实现文件的统一存储、智能解析、高效检索、安全分析和目标画像刻画，全面提升红方业务的信息化水平。

### 1.2 产品定位

网络安全红方文件汇聚平台是一个面向网络安全红方团队的专业文件管理平台，定位于为红方业务提供全生命周期的文件管理和分析服务。平台采用分布式架构设计，支持海量文件存储和高并发访问，通过人工智能技术实现智能化文件分析和语义检索，帮助红方团队提升工作效率和情报分析能力。

## 3. 功能需求详细说明

### 3.1 文件上传模块

#### 3.1.1 模块概述

文件上传模块是平台的核心入口，负责接收用户上传的各类文件，支持大文件分片上传、批量上传、断点续传和秒传功能，确保文件上传的高效性和可靠性。该模块需要处理每日十万加的文件上传请求，支持最高一千QPS的并发上传能力，同时保证大文件上传的成功率和稳定性。

#### 3.1.2 功能列表

| 功能编号 | 功能名称 | 优先级 | 功能描述 |
|---------|---------|-------|---------|
| F3.1.1 | 单文件上传 | P0 | 支持单个文件上传，包括文件选择、上传进度显示、上传结果反馈 |
| F3.1.2 | 大文件分片上传 | P0 | 支持大文件（大于一百MB）分片上传，提高上传成功率和速度 |
| F3.1.3 | 批量上传 | P0 | 支持多文件批量上传，支持文件夹整体上传 |
| F3.1.4 | 断点续传 | P0 | 支持上传中断后从断点继续上传，避免重复传输 |
| F3.1.5 | 秒传 | P1 | 通过文件哈希值判断文件是否已存在，实现秒传功能 |
| F3.1.6 | 上传限流 | P0 | 支持上传速率限制，防止占用过多带宽资源 |
| F3.1.7 | 文件校验 | P0 | 上传前进行文件类型、大小、格式校验 |
| F3.1.8 | 元数据提取 | P0 | 上传时自动提取文件基本信息（名称、大小、类型、哈希值等） |
| F3.1.9 | 标签设置 | P1 | 上传时支持设置文件标签，便于后续分类管理 |
| F3.1.10 | 上传记录 | P1 | 记录上传历史，支持查看上传日志和统计 |

#### 3.1.3 详细功能说明

##### F3.1.1 单文件上传

**功能描述：**

用户通过Web界面或API上传单个文件，系统接收文件并存储到对象存储系统，同时记录文件元数据信息。单文件上传是整个上传模块的基础功能，其他上传功能均基于此功能扩展实现。

**业务规则：**

| 规则编号 | 规则描述 |
|---------|---------|
| BR3.1.1.1 | 单个文件大小不超过五十GB |
| BR3.1.1.2 | 支持的文件类型包括：文档类（PDF、Word、Excel、PPT、TXT）、压缩包（ZIP、RAR、7z、TAR）、镜像文件（ISO、IMG）、流量文件（PCAP、PCAPNG）、日志文件（LOG、JSON、XML）、代码文件（各类编程语言源码）、配置文件（CONF、INI、YAML）等 |
| BR3.1.1.3 | 文件名长度不超过二百五十五个字符，不支持特殊字符（斜杠、反斜杠、冒号、星号、问号、引号、小于号、大于号、竖线） |
| BR3.1.1.4 | 上传前需进行用户身份认证和权限校验 |
| BR3.1.1.5 | 上传完成后自动计算文件哈希值（MD5、SHA256） |

**输入输出：**

| 类型 | 内容 |
|-----|------|
| 输入 | 文件内容、文件名、目标目录、标签（可选）、描述（可选） |
| 输出 | 文件ID、上传状态、存储路径、哈希值 |

**异常处理：**

| 异常场景 | 处理方式 |
|---------|---------|
| 文件类型不支持 | 返回错误提示，列出支持的文件类型 |
| 文件大小超限 | 返回错误提示，建议使用分片上传 |
| 存储空间不足 | 返回错误提示，联系管理员扩容 |
| 网络中断 | 支持断点续传，提示用户继续上传 |
| 权限不足 | 返回权限错误，提示联系管理员 |

##### F3.1.2 大文件分片上传

**功能描述：**

对于大文件（大于一百MB），系统自动采用分片上传方式，将文件分割成多个小块并发上传，提高上传成功率和速度。分片上传可以有效解决大文件上传失败的问题，同时支持断点续传功能。

**业务规则：**

| 规则编号 | 规则描述 |
|---------|---------|
| BR3.1.2.1 | 文件大小超过一百MB自动启用分片上传 |
| BR3.1.2.2 | 默认分片大小为十MB，可根据网络情况动态调整 |
| BR3.1.2.3 | 每个分片独立计算哈希值，用于完整性校验 |
| BR3.1.2.4 | 支持分片并发上传，默认并发数为四 |
| BR3.1.2.5 | 所有分片上传完成后，服务端进行文件合并 |
| BR3.1.2.6 | 合并完成后校验整体文件哈希值，确保完整性 |

**分片上传流程：**

```
第一步：初始化上传
客户端发送文件信息（文件名、大小、哈希值）
服务端返回上传ID和分片大小

第二步：分片上传（并发）
客户端将文件分割成多个分片
并发上传各分片，每个分片携带序号和哈希值
服务端接收分片并存储，返回确认信息

第三步：完成上传
客户端发送完成请求
服务端合并所有分片
校验整体哈希值，返回最终结果
```

##### F3.1.3 批量上传

**功能描述：**

支持用户一次性上传多个文件或整个文件夹，系统自动处理文件队列，提供整体进度显示。批量上传功能可以大幅提升用户的工作效率，特别适用于需要同时上传大量测试报告和扫描结果的场景。

**业务规则：**

| 规则编号 | 规则描述 |
|---------|---------|
| BR3.1.3.1 | 单次批量上传文件数量不超过一千个 |
| BR3.1.3.2 | 单次批量上传总大小不超过一百GB |
| BR3.1.3.3 | 支持文件夹结构保留，自动创建对应目录结构 |
| BR3.1.3.4 | 批量上传支持暂停、继续、取消操作 |
| BR3.1.3.5 | 单个文件上传失败不影响其他文件继续上传 |
| BR3.1.3.6 | 批量上传完成后生成上传报告，列出成功和失败的文件 |

##### F3.1.4 断点续传

**功能描述：**

当上传过程中断（网络故障、客户端崩溃等），用户可以在恢复后继续上传，避免从头开始。断点续传功能对于大文件上传尤为重要，可以显著提升用户的使用体验。

**业务规则：**

| 规则编号 | 规则描述 |
|---------|---------|
| BR3.1.4.1 | 上传中断后，服务端保留已上传的分片二十四小时 |
| BR3.1.4.2 | 客户端恢复后，查询已上传分片列表，继续上传未完成的分片 |
| BR3.1.4.3 | 分片上传记录存储在Redis中，设置过期时间 |
| BR3.1.4.4 | 支持跨设备续传（同一用户不同终端） |
| BR3.1.4.5 | 续传时需验证文件哈希值，防止文件被篡改 |

##### F3.1.5 秒传

**功能描述：**

上传前计算文件哈希值，如果系统中已存在相同哈希值的文件，则直接建立引用关系，无需重复上传文件内容。秒传功能可以大幅提升重复文件的处理效率，节省存储空间和带宽资源。

**业务规则：**

| 规则编号 | 规则描述 |
|---------|---------|
| BR3.1.5.1 | 秒传基于文件SHA256哈希值判断 |
| BR3.1.5.2 | 相同哈希值的文件共享同一份物理存储 |

#### 3.2.3 详细功能说明

##### F3.2.1 格式识别

**功能描述：**

通过文件魔数（Magic Number）和内容特征识别文件的真实格式，防止用户通过修改扩展名伪装文件类型。格式识别是文件解析的第一步，只有正确识别文件格式，才能选择正确的解析器进行处理。

| BR3.2.1.3 | 识别结果与扩展名不一致时，记录警告日志 |
| BR3.2.1.4 | 无法识别的格式标记为“未知”，仍允许存储 |

```

    |
    v
路由到对应解析器
    |
    +---> 文档解析器 -----> 提取文本、表格、图片
    +---> 压缩包解析器 ---> 提取文件列表、递归解析
    +---> 流量解析器 -----> 提取会话、协议信息
    +---> 日志解析器 -----> 提取结构化日志
    +---> 配置解析器 -----> 提取配置项
    +---> 图片解析器 -----> OCR识别
    |
    v
实体识别 -----> 提取IP、域名、哈希等实体
    |
    v
向量化处理 -----> 生成语义向量
    |
    v
存储结果
    +---> PostgreSQL (元数据、实体)
    +---> Elasticsearch (全文索引)
    +---> Milvus (向量索引)
    +---> Neo4j (实体关系)
```

**业务规则：**

**融合算法：**

```
综合得分 = w1 乘以 全文检索得分 + w2 乘以 向量检索得分 + w3 乘以 元数据匹配得分

其中：
- w1 + w2 + w3 = 1
- 默认 w1 = 0.4, w2 = 0.4, w3 = 0.2
- 各项得分归一到 零到一 区间
```

#### 3.4.4 分析流程

```
文件解析完成
    |
    v
路由到分析器
    |
    +---> 恶意检测引擎 -----> YARA/ClamAV/ML
    +---> 漏洞提取引擎 -----> 正则匹配/NER
    +---> 敏感检测引擎 -----> 正则匹配/规则引擎
    +---> 关联分析引擎 -----> 向量相似度/实体关联
    +---> 威胁情报引擎 -----> 外部API查询
    |
    v
结果汇总
    |
    v
生成分析报告
    |
    v
存储结果
    +---> PostgreSQL (分析结果)
    +---> Neo4j (关联关系)
    +---> Elasticsearch (索引)
```

### 5.4 用例关系描述

#### 5.4.1 文件上传用例关系

**前置条件：**

- 用户已登录系统
- 用户具有文件上传权限

**主成功场景：**

第一步，用户选择要上传的文件。

第二步，系统验证文件类型和大小。

第三步，系统计算文件哈希值。

第四步，系统检查是否支持秒传。

第五步，如需上传，系统初始化上传任务。

第六步，用户上传文件（支持分片上传）。

第七步，系统存储文件并记录元数据。

第八步，系统触发文件解析任务。

第九步，系统返回上传成功结果。

**扩展场景：**

第三步第一步，文件已存在（秒传）：系统建立文件引用关系，返回秒传成功结果。

第四步第一步，文件类型不支持：系统返回错误提示。

第六步第一步，上传中断：系统保存已上传分片，用户恢复上传时继续。

**后置条件：**

- 文件存储到对象存储
- 文件元数据记录到数据库
- 文件解析任务已创建

#### 5.4.2 文件检索用例关系

**前置条件：**

- 用户已登录系统
- 用户具有文件检索权限

**主成功场景：**

第一步，用户输入检索关键词或条件。

第二步，系统解析检索条件。

第三步，系统执行全文检索。

第四步，系统执行向量语义检索。

第五步，系统执行元数据检索。

第六步，系统合并检索结果。

第七步，系统对结果排序和分页。

第八步，系统返回检索结果列表。

第九步，用户查看检索结果。

**扩展场景：**

第二步第一步，高级检索：用户输入复杂检索条件，系统解析高级检索语法。

第三步第一步，无匹配结果：系统返回空结果列表，系统提供搜索建议。

第八步第一步，结果过滤：用户选择过滤条件，系统重新筛选结果。

**后置条件：**

- 检索历史已记录
- 检索统计已更新

#### 5.4.3 目标画像用例关系

**前置条件：**

- 用户已登录系统
- 用户具有目标查看权限
- 目标已创建

**主成功场景：**

第一步，用户选择目标。

第二步，系统加载目标基本信息。

第三步，系统加载目标资产画像。

第四步，系统加载目标漏洞画像。

第五步，系统加载目标人员画像。

第六步，系统加载目标组织画像。

第七步，系统加载目标关系图谱。

第八步，系统展示完整画像信息。

第九步，用户浏览画像详情。

**扩展场景：**

第三步第一步，资产信息为空：系统提示暂无资产信息。

第七步第一步，关系图谱过大：系统分页加载图谱节点。

第九步第一步，用户点击图谱节点：系统展示节点详情，系统高亮关联节点。

**后置条件：**

- 画像查看记录已保存
