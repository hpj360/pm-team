## 1. 设计原则

### 1.1 视觉风格定位

本平台面向网络安全红方专业人员，视觉风格需体现**专业、高效、安全、技术**四大核心特征。

#### 1.1.1 专业性

- 采用深色主题为主，营造专业、严肃的工作氛围
- 界面布局清晰有序，信息层次分明
- 避免过度装饰，保持界面简洁专业
- 数据展示精确、直观，便于专业人员快速获取信息

#### 1.1.2 高效性

- 优化操作流程，减少用户点击次数
- 重要操作入口明显，次要功能合理收纳
- 支持快捷键操作，提升专业用户效率
- 表格、列表支持批量操作，提高工作效率

#### 1.1.3 安全性

- 采用稳重的色彩体系，传递安全可信赖的感觉
- 敏感操作需二次确认，防止误操作
- 权限状态清晰可见，用户始终了解自己的权限范围
- 安全提示醒目但不突兀，不打断用户工作流

#### 1.1.4 技术感

- 采用科技感的配色方案（深蓝、青色、霓虹绿）
- 数据可视化采用专业图表风格
- 代码、日志等内容使用等宽字体展示
- 界面元素带有适度的技术感细节（边框、阴影、渐变）

### 1.2 用户体验原则

#### 1.2.1 一致性原则

| 维度 | 说明 |
|-----|------|
| 视觉一致 | 相同功能的组件保持视觉样式一致 |
| 交互一致 | 相同操作行为保持交互方式一致 |
| 文案一致 | 相同概念使用统一术语 |
| 布局一致 | 相同类型页面保持布局结构一致 |

#### 1.2.2 反馈原则

| 场景 | 反馈方式 |
|-----|---------|
| 操作成功 | 轻量级成功提示，自动消失 |
| 操作失败 | 明确的错误信息，提供解决方案 |
| 加载中 | 加载动画或进度条 |
| 操作确认 | 二次确认弹窗 |
| 状态变化 | 实时更新状态显示 |

#### 1.2.3 容错原则

- 危险操作（删除、清空等）需二次确认
- 提供撤销功能，支持操作回退
- 表单输入提供实时校验和友好提示
- 错误信息清晰明确，指导用户解决问题

#### 1.2.4 效率原则

- 支持键盘快捷键操作
- 常用功能支持收藏和快捷访问
- 搜索支持历史记录和智能建议
- 列表支持多选和批量操作

#### 1.2.5 可访问性原则

- 色彩对比度符合WCAG 2.1 AA标准（至少4.5:1）
- 所有交互元素支持键盘访问
- 图片提供替代文本
- 表单元素关联标签

## 3. 字体体系

### 3.1 字体家族

#### 3.1.1 中文字体

| 字体名称 | 使用场景 | 备选字体 |
|---------|---------|---------|

| 思源宋体 | 长文本阅读 | Noto Serif SC, serif |

**字体栈定义：**

```css
/* 主要UI字体 */
font-family: "Source Han Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif;

/* 品牌字体 */
font-family: "Alibaba PuHuiTi", "Source Han Sans SC", sans-serif;

/* 阅读字体 */
```

#### 6.1.2 设计原则

- **简洁性**：去除不必要的装饰，保持图标简洁
